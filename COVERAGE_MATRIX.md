# AdminAvenger Coverage Matrix

This file describes verified repository coverage.

It is not a marketing promises list. "Implemented" means there is supporting
source code and/or tests in the repository. Planned categories must not be
presented as live product coverage.

| Field | Meaning |
| --- | --- |
| Engine/category | Stable identifier or descriptive name |
| User problem | What material it is intended to handle |
| Current maturity | Stable / implemented / narrow / beta / demo / fallback / planned |
| Input examples | Typical letters, emails, bills or documents |
| Classification signals | Key phrases or document clues |
| Confusion risks | Similar categories it could be mistaken for |
| Facts extracted | Dates, amounts, references, provider, requirements, etc. |
| Evidence expected | Documents or facts the user should check |
| Current output | Explanation, checklist, draft, case, etc. |
| Forbidden output | Decisions or claims it must not make |
| Automated tests | Relevant test files or fixture names |
| Live tested | Yes/no and test name |
| Next hardening step | What remains before it is considered stable |

## F. Coverage Gaps

- Incomplete golden fixtures for every implemented engine and edge case.
- Limited cross-engine confusion tests outside the main golden corpus.
- Jurisdiction handling is not systematic.
- Source review dates are not embedded in all trusted guidance or engine copy.
- Accessibility testing is not complete.
- Mobile camera/OCR testing remains dependent on manual live checks.
- OCR error fixtures need expansion, especially for realistic blurry/cropped
  letters and scanned PDFs.
- Hosted security model is not designed because hosted AI/backend are not
  implemented.
- Account-backed privacy model is not designed because accounts/database are
  not implemented.
- No import/restore flow for local backup exports.
- No redaction preview for adviser/case exports.
- Workplace and Community Helper remain controlled betas and should not be
  auto-routed through normal Home analysis.
- Generic mock findings should not be treated as complete specialist engines.
